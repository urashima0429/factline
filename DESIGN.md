# Factorio 生産ライン縦断面レイアウト最適化ツール

## 目的

Factorio の生産ラインを制約充足問題 (CSP) + 最適化として定式化し，バスを左側に置いたレイアウトで **幅最小化** を行う．高さは機械種別ごとに固定値とする．

---

# Model

- 盤面はタイル単位の格子 (grid) として表現される．
- 機械は 3x3 または 4x4 タイル．
- モジュール範囲化技術（範囲内の機械に速度等のバフを与える）を考慮し，上/下2行のみを入出力に利用できるものとする
- 結果として，盤面高さは機械種別で固定（3x3: 7, 4x4: 8）．
- ベルトは向き (`Direction`) とレーン (`Lane`: LEFT/RIGHT) を持つ．インサータは向きのみを持つ．

--- 

# Formulation
- Objective function: 盤面の幅を最小化（高さは固定）
- Constraints: 
  - C1. 機械入力成立: 各機械の各入力アイテムは「ベルト」「箱」「前段機械」のいずれか1つから，インサータ経由で供給される
  - C2. 機械出力成立: 各機械の出力アイテムは「ベルト」「箱」「組立機」のいずれか1つへインサータ経由で排出される．
  - C3. I/O行の整合: 入力ラベル行は上側ベルト行，出力ラベル行は下側ベルト行に一致
  - C4. レーン重複禁止: I/O ラベルの `(row, lane)` は全て異なる
  - C5. 逆方向同一行禁止: `>` と `<` の I/O は同一タイル行を共有しない
- Input 
  - ハブから生産ラインへ入力されるアイテム集合
  - 箱から生産ラインへ入力されるアイテム集合
  - 生産ラインからハブへ出力されるアイテム
  - 生産ラインから箱に格納されるアイテム
  - 生産ラインに配置される組立機の数と仕様（固形入力アイテム集合，流体入力0/1，出力アイテム1）
- Output
  - もし存在しない場合は **UNSAT** を出力．
  - もし存在する場合は最小幅レイアウト（2x2描画＋左ラベル）．

## 制約と実装関数の対応

- C1/C2（機械I/O成立）
  - 実装: `solver.py::_add_machine_io_flow_constraints`
  - 補足: `via_belt_{src,dst,item}` Bool を使って「機械→ベルト→機械」の中継接続を表現
- C3/C4/C5（I/O行・レーン整合）
  - 実装: `solver.py::_add_io_lane_constraints`
  - 補足: `row` と `lane` から `slot=row*2+lane` を作り `AllDifferent`
- 目的関数（幅最小）
  - 実装: `solver.py::solve_first_feasible` 内 `model.Minimize(width)`
- 解の可視化（方向/レーンを自然言語ラベルに反映）
  - 実装: `render.py::_build_line_labels`, `render.py::_line_index`, `render.py::render_solution`
